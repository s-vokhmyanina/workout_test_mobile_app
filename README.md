В проекте есть основные слои:
- domain
  - interactors : здесь лежат интеракторы (p.s. можно было бы разбить на UseCase, но я решила оставить 1 интерактор на 1 экран)
  - repositories : здесь лежит репозиторий для походов в интернет
  - network : здесь логика для работы с интернетом, api и т.д.
  - models : модели dto
  - etc
- presentation (пока только ui папка)
  - ui
    - для каждого экрана своя папка
      - state
      - viewmodel
      - fragment
      - adapter (если есть)
- di (dagger для этого карапуза подключать не стала, свои Factory есть)


Есть еще вспомогательные классы, свои ошибки BaseError и Response (аналог Result)

- логику с плеером вынесла бы отдельно, а не во фрагменте
- сделала бы вью модели более полезными
- если бы позволял api добавила бы пейджинг для списка
- для навигации тоже бы какие-нибудь роутеры может хотя бы как интерфейс прописать
